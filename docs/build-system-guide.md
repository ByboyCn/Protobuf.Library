# Protobuf Library - 构建系统指南

## 🎯 功能概述

Protobuf.Library 提供了完整的构建时配置系统，支持：

- ✅ **通配符匹配**: `**/*.proto`, `Protos/**/*.proto` 等
- ✅ **多目录支持**: 从不同目录包含 proto 文件
- ✅ **自动导入**: 无需手动配置，开箱即用
- ✅ **构建集成**: 编译时自动生成代码
- ✅ **VS 支持**: 完整的 IntelliSense 和错误提示
- ✅ **AOT 友好**: 完全支持 NativeAOT 编译
- ✅ **反射 API**: 自动生成消息描述符

## 🚀 快速开始

### 最简单的配置

只需在你的 `.csproj` 文件中添加：

```xml
<ItemGroup>
  <!-- 核心库 -->
  <ProjectReference Include="Protobuf.Core\Protobuf.Core.csproj" />
  
  <!-- Source Generator -->
  <ProjectReference Include="Protobuf.Generator\Protobuf.Generator.csproj"
                    OutputItemType="Analyzer"
                    ReferenceOutputAssembly="false" />
  
  <!-- 可选：JSON 支持 -->
  <ProjectReference Include="Protobuf.Json\Protobuf.Json.csproj" />
  
  <!-- 可选：反射 API -->
  <ProjectReference Include="Protobuf.Reflection\Protobuf.Reflection.csproj" />

  <!-- 包含所有 proto 文件 -->
  <ProtoFile Include="**\*.proto" />
</ItemGroup>
```

### 构建项目

```bash
dotnet build
```

✅ Source Generator 会自动为每个 `.proto` 文件生成对应的 C# 代码！

## 📁 支持的配置模式

### 1. 通配符匹配

```xml
<ItemGroup>
  <!-- 所有子目录 -->
  <ProtoFile Include="**\*.proto" />
  
  <!-- 特定目录 -->
  <ProtoFile Include="Protos\**\*.proto" />
  
  <!-- 多个模式 -->
  <ProtoFile Include="Api\*.proto" />
  <ProtoFile Include="Shared\**\*.proto" />
  
  <!-- 递归匹配 -->
  <ProtoFile Include="Messages\**\*.proto" />
</ItemGroup>
```

### 2. 排除文件

```xml
<ItemGroup>
  <!-- 包含所有，排除测试文件 -->
  <ProtoFile Include="**\*.proto" Exclude="**\*.test.proto" />
  
  <!-- 排除特定目录 -->
  <ProtoFile Include="Protos\**\*.proto" Exclude="Protos\Deprecated\**\*.proto" />
  
  <!-- 多个排除模式 -->
  <ProtoFile Include="**\*.proto" Exclude="**\*.test.proto;**\*.tmp.proto" />
</ItemGroup>
```

### 3. 精确控制

```xml
<ItemGroup>
  <!-- 只包含特定文件 -->
  <ProtoFile Include="Protos\api.proto" />
  <ProtoFile Include="Protos\common.proto" />
  <ProtoFile Include="Protos\models\user.proto" />
</ItemGroup>
```

### 4. 跨项目共享

```xml
<ItemGroup>
  <!-- 来自其他项目的 proto 文件 -->
  <ProtoFile Include="..\SharedProtos\**\*.proto" />
  
  <!-- 解决方案目录 -->
  <ProtoFile Include="$(SolutionDir)Protos\**\*.proto" />
</ItemGroup>
```

## 🔧 高级配置

### 项目属性

```xml
<PropertyGroup>
  <!-- 启用扩展分析器规则（推荐） -->
  <EnforceExtendedAnalyzerRules>true</EnforceExtendedAnalyzerRules>
  
  <!-- 生成器输出详细程度 -->
  <ProtobufGeneratorVerbosity>detailed</ProtobufGeneratorVerbosity>
  
  <!-- 目标框架 -->
  <TargetFramework>net8.0</TargetFramework>
  
  <!-- 语言版本 -->
  <LangVersion>latest</LangVersion>
  
  <!-- 可空引用类型 -->
  <Nullable>enable</Nullable>
</PropertyGroup>
```

### 条件编译

```xml
<ItemGroup>
  <!-- 只在特定配置中包含 -->
  <ProtoFile Include="Protos\debug.proto" Condition="'$(Configuration)'=='Debug'" />
  
  <!-- 只在特定平台中包含 -->
  <ProtoFile Include="Protos\windows.proto" Condition="'$(TargetFramework)'=='net8.0-windows'" />
</ItemGroup>
```

### 自定义输出目录

```xml
<ItemGroup>
  <!-- 生成的代码会放在 obj/Generated/ 目录 -->
  <ProtoFile Include="Protos\**\*.proto">
    <!-- 可选：自定义元数据 -->
    <Visible>false</Visible>
  </ProtoFile>
</ItemGroup>
```

## 🛠️ Source Generator 工作原理

### 构建时执行流程

1. **项目加载**: MSBuild 加载 `.csproj` 文件
2. **收集文件**: 查找所有 `<ProtoFile>` 项
3. **启动生成器**: Protobuf.Generator 作为分析器运行
4. **解析文件**: 解析每个 `.proto` 文件
5. **生成代码**: 为每个 message/enum 生成 C# 代码
6. **编译集成**: 生成的代码参与编译

### 生成的代码结构

对于 `Person.proto`:

```protobuf
syntax = "proto3";

package Example;

message Person {
  string name = 1;
  int32 id = 2;
}
```

生成 `Person.g.cs`:

```csharp
// <auto-generated>
//     Generated by Protobuf Source Generator
// </auto-generated>

#pragma warning disable CS1591

namespace Example
{
    public partial class Person : IMessage<Person>
    {
        private string _name = "";
        private int _id = 0;
        
        public bool HasName => _name != "";
        public bool HasId => _id != 0;
        
        public string Name => _name;
        public int Id => _id;
        
        public void SetName(string value) { _name = value ?? ""; }
        public void SetId(int value) { _id = value; }
        
        public void ClearName() { _name = ""; }
        public void ClearId() { _id = 0; }
        
        // ... 更多生成的方法
        
        public static MessageParser<Person> Parser { get; } = 
            new MessageParser<Person>(() => new Person());
            
        public static MessageDescriptor Descriptor { get; }
        
        public static IEnumerable<IFieldAccessor> GetFieldAccessors()
    }
}
```

## 🐛 故障排除

### 常见问题

#### 1. 生成的代码没有 IntelliSense

**解决方案**: 确保引用了核心库和生成器

```xml
<ProjectReference Include="Protobuf.Core\Protobuf.Core.csproj" />
<ProjectReference Include="Protobuf.Generator\Protobuf.Generator.csproj"
                    OutputItemType="Analyzer"
                    ReferenceOutputAssembly="false" />
```

#### 2. 编译错误：找不到类型

**原因**: 生成器未能找到 `.proto` 文件

**解决方案**: 检查 `<ProtoFile>` 配置是否正确

```xml
<!-- 检查路径是否正确 -->
<ProtoFile Include="Protos\**\*.proto" />
```

#### 3. 生成器警告

```
warning RS1036: "Protobuf.Generator.ProtoSourceGenerator": 
包含分析器或源生成器的项目应指定属性 "<EnforceExtendedAnalyzerRules>true</EnforceExtendedAnalyzerRules>"
```

**解决方案**: 添加推荐属性

```xml
<PropertyGroup>
  <EnforceExtendedAnalyzerRules>true</EnforceExtendedAnalyzerRules>
</PropertyGroup>
```

#### 4. 生成器执行失败

**症状**: 生成器报告错误

**解决方案**: 检查 `.proto` 文件语法

```bash
# 使用解析器验证语法
cd examples/Examples
dotnet run -- parser your-file.proto
```

### 调试生成器

#### 启用详细输出

```xml
<PropertyGroup>
  <ProtobufGeneratorVerbosity>detailed</ProtobufGeneratorVerbosity>
</PropertyGroup>
```

#### 检查生成的文件

生成的文件位于:
```
obj/Debug/net8.0/
├── Protobuf.Generator.ProtobufSourceGenerator/
│   └── Person.g.cs
```

#### 验证生成的代码

```bash
# 查看生成的文件
cat obj/Debug/net8.0/Protobuf.Generator.ProtobufSourceGenerator/Person.g.cs
```

## 📊 构建性能

### 性能优化

#### 1. 增量构建

生成器只重新生成修改的 `.proto` 文件：

```bash
# 第一次构建（较慢）
dotnet build

# 后续构建（快速，只处理修改的文件）
dotnet build
```

#### 2. 并行处理

多个 `.proto` 文件并行处理：

```xml
<ItemGroup>
  <ProtoFile Include="Protos\**\*.proto" />
  <!-- 生成器会并行处理这些文件 -->
</ItemGroup>
```

#### 3. 内存优化

大型项目使用流式处理：

```xml
<ItemGroup>
  <!-- 对于大量文件，建议分批处理 -->
  <ProtoFile Include="Protos\Core\*.proto" />
  <ProtoFile Include="Protos\Api\*.proto" />
  <ProtoFile Include="Protos\Models\*.proto" />
</ItemGroup>
```

## 🎨 最佳实践

### 1. 项目组织

**推荐**: 按功能组织 `.proto` 文件

```
Protos/
├── Core/
│   ├── common.proto
│   └── base.proto
├── Api/
│   ├── requests.proto
│   └── responses.proto
└── Models/
    ├── user.proto
    └── product.proto
```

### 2. 命名规范

**推荐**: 使用 PascalCase 文件名和 Message 名称

```protobuf
// ✅ 好
message UserProfile { }
enum UserStatus { }

// ❌ 避免
message user_profile { }
enum user_status { }
```

### 3. 包命名

**推荐**: 使用公司/项目名称作为包前缀

```protobuf
syntax = "proto3";

package YourCompany.YourProject;

message MyMessage { }
```

### 4. 导入组织

**推荐**: 清晰的 import 结构

```protobuf
syntax = "proto3";

package Example;

// 导入基础类型
import "Protos/common.proto";

// 导入模型
import "Protos/models/user.proto";

// 当前文件定义
message ApiRequest { }
```
